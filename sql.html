1. Swiggy Customer Analysis

Description:

    I worked on an analytics project using SQL, focusing on data from a food ordering app. The objective was to study and analyze the order patterns within the app's dataset. Through a series of intricate SQL queries, I aimed to extract valuable insights to improve understanding and decision-making.
  • Finding the latest order placed by customers: Identified and analyzed the most recent orders placed by customers to understand recent ordering trends.
  • Outlet-wise analysis: Conducted detailed analyses to determine outlet-wise order counts, cumulative order counts, total bill amounts, and cumulative bill amounts. This provided insights into outlet performance and revenue generation.
  • Printing order summary: Generated concise order summaries to efficiently monitor order statuses and identify any discrepancies or issues.
  • Printing crosstab for customer-wise and outlet-wise total number of orders: Utilized crosstab queries to represent customer-wise and outlet-wise total numbers of orders, facilitating a comprehensive understanding of ordering patterns.
  • Printing crosstab for customer-wise and outlet-wise total sales: Similarly, utilized crosstab queries to represent customer-wise and outlet-wise total sales, providing insights into revenue generation across different customer segments and outlets.
    To perform this analysis, I employed various SQL techniques including Common Table Expressions (CTE), RANK() OVER(PARTITION BY), nested queries, CASE statements, IF functions, joins, GROUP BY, ORDER BY, etc. These techniques enabled me to extract meaningful insights and perform complex analyses on the dataset.

2. Data Cleaning Project

Description:

    This SQL project focuses on data preprocessing tasks for the 'housingdata' table in the 'project' database. The objective is to ensure data cleanliness and improve its structure for efficient analysis. Here's an overview of the tasks performed:
  • Populating null property address: The NULL values in the 'PropertyAddress' field are replaced with values from records with the same 'ParcelID', ensuring completeness of address information.
  • Splitting property address: The 'PropertyAddress' field is split into 'propertyaddress_area' and 'propertyaddress_city' to provide more granular location information for analysis.
  • Splitting owner address: Similar to the property address, the 'OwnerAddress' field is split into 'owneraddress_area', 'owneraddress_city', and 'owneraddress_state' to enhance address information.
  • Updating SoldAsVacant field: The 'SoldAsVacant' field values 'Y' and 'N' are standardized to 'YES' and 'NO' for consistency.
  • Removing duplicates: Duplicate records are removed based on specific criteria to ensure data integrity and avoid redundancy.
  • Deleting unused columns: Columns such as 'OwnerAddress', 'TaxDistrict', and the original 'PropertyAddress' are dropped from the table, streamlining the dataset for analysis.
    These SQL queries demonstrate a systematic approach to data preprocessing, essential for ensuring data quality and enhancing the effectiveness of subsequent analysis tasks on housing data.

3. World Analysis

Description:

    This SQL project is a comprehensive exploration of global data sourced from the 'world' database. The dataset includes essential fields such as continent, country name, population, life expectancy, year of independence, among others. Through a series of intricate SQL queries, the project aims to extract valuable insights and perform various analyses on the dataset.
  • Continent-wise population and country count: This query aggregates population and country count data by continent, offering an overview of global population distribution.
  • Countries with population above world average: Identifies countries with populations exceeding the global average, facilitating insights into regions of significant population density.
  • Countries with population above continent average: This query drills down further, identifying countries with populations surpassing the average within their respective continents, providing localized insights into population distribution.
  • Top countries based on life expectancy: Lists the top five countries globally based on life expectancy, highlighting regions with the highest life expectancy rates.
  • Continent-wise top countries based on life expectancy (using Common Table Expression): Utilizes a Common Table Expression (CTE) to identify the top three countries in life expectancy within each continent, offering continent-specific insights.
  • Joining country data with official languages: This query joins country data with official languages sourced from the 'countrylanguage' table, providing insights into linguistic diversity across continents.
  • Continent-wise cumulative population: Calculates the cumulative population for each continent, facilitating a deeper understanding of population trends at a continental level.
  • Median population and corresponding country: Identifies the median population globally and the corresponding country, offering insights into regions with moderate population densities.
  • Median population and corresponding country in each continent: Extends the previous analysis to identify median populations within each continent, providing localized insights into population distribution.
  • Crosstab analysis of population percentages: Utilizes temporary tables to calculate population percentages and creates a crosstab analysis categorizing countries based on population density.
  • Countries independent after India's independence: Lists countries that gained independence after India, offering historical insights into post-colonial nationhood.
    This project showcases the versatility of SQL in performing complex data analyses and extracting valuable insights from diverse datasets, essential for informed decision-making and understanding global trends.
